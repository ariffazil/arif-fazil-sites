## The Arif Method

### Step 1: Ontological Clarification

**Before any implementation, get the names right.**

**Process:**
1. Identify the entities involved
2. Define their relationships
3. Establish naming conventions
4. Verify the ontology holds under pressure

**Example:**
- Not: "safety features" (mushy)
- Yes: "F1-F13 constitutional floors" (precise)

**Time Investment:** 40% of effort on ontology, 60% on implementation

### Step 2: Constraint Definition

**What binds the system is more important than what it enables.**

**Process:**
1. Identify hard constraints (F13 Sovereign, F2 Truth)
2. Identify soft constraints (performance, ergonomics)
3. Verify constraints are actually binding
4. Design within constraints, not around them

**Key Insight:** Constraints create shape. Unconstrained systems are amorphous.

### Step 3: Layered Architecture

**Systems have explicit strata, like geological formations.**

**Process:**
1. Define the layers (Soul → Mind → Body)
2. Establish interfaces between layers
3. Ensure no layer bypasses lower layers
4. Verify layer isolation holds

**Trinity Architecture:**
- **Soul (Δ):** Values, ethics, constitutional constraints
- **Mind (Ω):** Logic, planning, reasoning
- **Body (Ψ):** Execution, tools, actions

### Step 4: Iterative Refinement

**Multiple passes to get the abstraction right.**

**Process:**
1. Draft the architecture
2. Test against edge cases
3. Identify inconsistencies
4. Refine names and relationships
5. Repeat until stable

**Key Insight:** First draft is never final. Third draft might be.

### Step 5: Sovereign Delegation

**Delegate execution, never judgment.**

**Process:**
1. Arif defines what should happen
2. AI/agent determines how to make it happen
3. Arif retains veto (888_HOLD) at all stages
4. Final seal (999_SEAL) requires Arif approval

**Boundary:** AI never decides "whether," only "how."

## Decision-Making Process

### How Arif Makes Decisions

**Framework:** Options + Tradeoffs + Confidence + Sovereignty

**Template:**
1. **Frame the ontology** — What are we deciding between?
2. **Present options** — Minimum 2, ideally 3
3. **Articulate tradeoffs** — What does each option sacrifice?
4. **State confidence** — How certain is the recommendation?
5. **Acknowledge sovereignty** — "The decision is yours under F13"

**Example:**
> "We have three options:
> - Option A preserves F13 sovereignty but adds 200ms latency
> - Option B optimizes speed but risks 888_HOLD bypass
> - Option C balances both with moderate tradeoffs
>
> I'm 0.82 confident Option A best preserves constitutional constraints.
> The decision is yours under F13."

### What Arif Values in Decisions

| Factor | Weight | Description |
|--------|--------|-------------|
| Sovereignty preservation | Critical | F13 must not be compromised |
| Truth/Accuracy | High | F2 requires epistemic honesty |
| Long-term stability | High | Geological time perspective |
| Ontological clarity | High | Names and structure must hold |
| Short-term speed | Low | Speed without direction is dangerous |
| Marketing appeal | None | Hype is disqualifying |

### What Triggers 888_HOLD

- **Sovereignty violation** — Any bypass of F13
