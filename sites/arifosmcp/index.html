// TRANSPORT

Connection Modes

Three transport modes available. Streamable HTTP is the current standard per MCP spec (SSE deprecated March 2025).

Streamable HTTP

RECOMMENDED · MCP SPEC 2025

The current MCP transport standard. Single endpoint accepts JSON-RPC POST requests. Responses may stream via HTTP for long-running tool calls. SSE transport is fully deprecated.

URL: https://arifosmcp.arif-fazil.com/mcp
Method: POST
Content-Type: application/json
# Optional: X-API-Key: <key>

STDIO

LOCAL · PROCESS

For local development and Claude Desktop integration. Spawns the runtime as a subprocess communicating via stdin/stdout. No network required.

python -m arifosmcp.runtime stdio

# Claude Desktop config:
{
  "mcpServers": {
    "arifos": {
      "command": "python",
      "args": ["-m", "arifosmcp.runtime",
               "stdio"]
    }
  }
}

WebMCP & API Endpoints

Web-accessible documentation and browser-friendly endpoints for exploration and integration.

  • POST /mcp Primary MCP JSON-RPC endpoint — Streamable HTTP transport
  • GET /health Runtime health check — returns status, version, uptime
  • GET /tools List all 11 registered tools with schemas
  • GET /version Runtime version, build hash, constitutional floor count
  • GET /webmcp Browser-accessible MCP docs and interactive tool explorer
  • GET /.well-known/agent.json A2A agent card — machine-readable agent identity and capabilities
  • POST /a2a A2A task endpoint — agent-to-agent task delegation
  • GET /llms.txt LLM-optimised context file — tool list, transport, governance summary
