// MCP PROTOCOL SPECIFICATION

MCP Spec

Complete protocol reference for connecting to and invoking tools on the arifOS MCP runtime. All calls pass through 13 constitutional floors before a verdict is returned.

Endpoint & Headers CONNECTION

The production MCP server listens at a single HTTP endpoint. All MCP JSON-RPC calls are issued as POST requests to this URL.

PropertyValue
Endpoint URLhttps://arifosmcp.arif-fazil.com/mcp
HTTP MethodPOST
Content-Typeapplication/json
Acceptapplication/json
X-API-Key<your-key> — optional, see Authorization
TransportStreamable HTTP (SSE deprecated March 2025)
Note: SSE (Server-Sent Events) transport was deprecated as of the MCP specification update in March 2025. All clients should use Streamable HTTP.

Content-Type HEADERS

All requests must include the correct content-type header. The server rejects malformed or missing content-type with HTTP 415.

POST /mcp HTTP/1.1
Host: arifosmcp.arif-fazil.com
Content-Type: application/json
Accept: application/json
X-API-Key: <optional>

{ ... JSON-RPC body ... }

Authorization AUTH

Authentication is optional by default. When the server has COPILOT_API_KEY set in its environment, all requests require a matching X-API-Key header.

X-API-Key

X-API-Key: sk-arifos-xxxxxxxxxxxxxxxxxxxxxxxx

ARIFOS_GOVERNANCE_SECRET

High-governance operations — those triggering F13 Sovereign review — may additionally require the governance secret header. This is reserved for administrative and override operations.

X-Governance-Secret: <ARIFOS_GOVERNANCE_SECRET value>
Warning: Never embed API keys or governance secrets in client-side code. Pass them via environment variables or a secrets manager.
HeaderRequiredDescription
X-API-Key optional Bearer key. Required only when COPILOT_API_KEY env is set on server.
X-Governance-Secret optional Required for F13 Sovereign override operations only.

Tool Invocation JSON-RPC 2.0

All tool calls follow JSON-RPC 2.0 format. The method is always tools/call. The params object contains the tool name and its arguments.

Request Structure

{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "<tool_name>",
    "arguments": {
      // Tool-specific arguments
    }
  },
  "id": "<request-id>"
}
FieldTypeRequiredDescription
jsonrpc string required Must be "2.0"
method string required Must be "tools/call"
params.name string required Registered tool name (e.g. init_anchor)
params.arguments object required Tool-specific argument map
id string | number required Client-assigned request identifier; echoed in response

List Tools

To discover available tools and their schemas, send a tools/list request:

{
  "jsonrpc": "2.0",
  "method": "tools/list",
  "params": {},
  "id": 1
}

Response Schema — ToolEnvelope SCHEMA

All tool responses are wrapped in a ToolEnvelope. The envelope carries the JSON-RPC frame, the constitutional verdict, floor scores, and the tool-specific result payload.

{
  "jsonrpc": "2.0",
  "id": "<request-id>",
  "result": {
    "content": [
      {
        "type": "text",
        "text": "<tool output as string or JSON>"
      }
    ],
    "envelope": {
      "verdict":         "SEAL",           // SEAL | SABAR | VOID | 888_HOLD
      "floor":           13,               // Last floor reached (1–13)
      "constitutional_score": 0.97,        // 0.0–1.0
      "session_token":   "tok_xxx",        // From init_anchor
      "stage":           "999_VAULT",      // Pipeline stage
      "timestamp_utc":   "2026-03-24T...", // ISO 8601
      "flags":           [],               // Any triggered flags
      "hold_reason":     null              // Populated on 888_HOLD
    }
  }
}
FieldTypeDescription
verdict string Constitutional verdict: SEAL, SABAR, VOID, or 888_HOLD
floor integer Last constitutional floor reached (1–13)
constitutional_score number Aggregate constitutional compliance score (0.0–1.0)
session_token string Session token from init_anchor (required for traceability)
stage string Pipeline stage where response was generated
timestamp_utc string ISO 8601 UTC timestamp of verdict issuance
flags array Triggered constitutional flags (empty array if none)
hold_reason string | null Human-readable reason for 888_HOLD escalation; null otherwise

Error Contracts ERRORS

Errors follow JSON-RPC 2.0 error format. The code field maps to standard JSON-RPC codes plus arifOS-specific extension codes.

{
  "jsonrpc": "2.0",
  "id": "<request-id>",
  "error": {
    "code": -32602,
    "message": "Invalid params",
    "data": {
      "field":   "session_token",
      "reason":  "Missing required field. Call init_anchor first.",
      "verdict": "VOID"
    }
  }
}
CodeNameDescription
-32700Parse errorRequest body is not valid JSON
-32600Invalid requestJSON-RPC structure is invalid
-32601Method not foundUnknown method or tool name
-32602Invalid paramsMissing or malformed tool arguments
-32603Internal errorRuntime execution failure
-32001Auth requiredX-API-Key missing or invalid
-32002Session missinginit_anchor must be called first
-32003888_HOLDF13 Sovereign hold — awaiting human confirmation
-32004VOID verdictConstitutional floor rejected the request

Verdict System GOVERNANCE

Every tool call is assigned a verdict by the constitutional floor pipeline. The verdict appears in the envelope.verdict field of every response.

SEAL
All 13 floors passed. Response is constitutionally cleared. Trust the output.
SABAR
Partial pass. Response valid but additional evidence or delay recommended before acting.
VOID
Floor rejection. Request did not meet constitutional requirements. Do not act on output.
888_HOLD
F13 Sovereign escalation. High-stakes action blocked pending human confirmation. Check hold_reason.
888_HOLD behaviour: When a HOLD is returned, the tool call is queued and not executed. The client must obtain human sign-off (via the governance endpoint or out-of-band) and re-submit with the confirmation token.

Streamable HTTP Transport RECOMMENDED

The default and recommended transport. All tool calls are POST requests. For long-running tools (e.g. arifOS_kernel), the server may stream partial results using HTTP chunked transfer encoding.

Request

POST https://arifosmcp.arif-fazil.com/mcp
Content-Type: application/json
Accept: application/json, text/event-stream

{ ... JSON-RPC body ... }

Streaming Response (long tools)

HTTP/1.1 200 OK
Content-Type: text/event-stream

data: {"type":"progress","stage":"111_SENSE","pct":20}
data: {"type":"progress","stage":"333_EXPLORE","pct":55}
data: {"type":"progress","stage":"888_JUDGE","pct":90}
data: {"type":"result","jsonrpc":"2.0","result":{...}}
SSE Deprecation: The old SSE-only transport (separate /sse endpoint) is fully deprecated as of MCP spec March 2025. The new Streamable HTTP transport uses the single /mcp endpoint for both synchronous and streaming responses.

STDIO Transport LOCAL

For local development and Claude Desktop integration. The runtime is spawned as a subprocess. JSON-RPC messages are exchanged via stdin/stdout with newline delimiters.

Start

python -m arifosmcp.runtime stdio

Claude Desktop Configuration

// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "arifos": {
      "command": "python",
      "args": ["-m", "arifosmcp.runtime", "stdio"]
    }
  }
}

STDIO Message Format

// Each message is a single JSON line followed by \n
→ stdin:  {"jsonrpc":"2.0","method":"tools/call","params":{...},"id":1}\n
← stdout: {"jsonrpc":"2.0","result":{...},"id":1}\n

Example — init_anchor 000_INIT

Always the first call. Forges a cryptographic session token that must be passed to all subsequent tool calls for traceability and constitutional audit.

Request

POST https://arifosmcp.arif-fazil.com/mcp
Content-Type: application/json

{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "init_anchor",
    "arguments": {
      "session_id": "session-2026-03-24-001",
      "intent": "Evaluating autonomous vehicle safety policy",
      "agent_id": "my-agent@example.com",
      "metadata": {
        "environment": "production",
        "client": "custom-mcp-client-v1.0"
      }
    }
  },
  "id": "init-001"
}

Response

{
  "jsonrpc": "2.0",
  "id": "init-001",
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"session_token\":\"tok_2026_abc123xyz\",\"expires_utc\":\"2026-03-25T06:32:00Z\"}"
      }
    ],
    "envelope": {
      "verdict": "SEAL",
      "floor": 1,
      "constitutional_score": 1.0,
      "session_token": "tok_2026_abc123xyz",
      "stage": "000_INIT",
      "timestamp_utc": "2026-03-24T06:32:00Z",
      "flags": [],
      "hold_reason": null
    }
  }
}

Example — arifOS_kernel 444_ROUTER

The full pipeline orchestrator. Routes the query through all stages 000→999 and returns a complete constitutional analysis with verdict, evidence, and reasoning chain.

Request

POST https://arifosmcp.arif-fazil.com/mcp
Content-Type: application/json

{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "arifOS_kernel",
    "arguments": {
      "session_token": "tok_2026_abc123xyz",
      "query": "Should this autonomous vehicle override a red light to avoid a collision?",
      "context": {
        "urgency": "high",
        "stakeholders": ["driver", "pedestrians", "other_vehicles"],
        "jurisdiction": "Malaysia"
      },
      "pipeline_mode": "full",
      "require_evidence": true
    }
  },
  "id": "kernel-001"
}

Response (abbreviated)

{
  "jsonrpc": "2.0",
  "id": "kernel-001",
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"analysis\":{\"verdict\":\"SEAL\",\"reasoning\":\"...\",\"evidence\":[...],\"constitutional_score\":0.91,\"recommendation\":\"...\"}}"
      }
    ],
    "envelope": {
      "verdict": "SEAL",
      "floor": 13,
      "constitutional_score": 0.91,
      "session_token": "tok_2026_abc123xyz",
      "stage": "999_VAULT",
      "timestamp_utc": "2026-03-24T06:32:14Z",
      "flags": ["HIGH_STAKES", "MULTI_STAKEHOLDER"],
      "hold_reason": null
    }
  }
}
Vault sealed: This call was automatically logged to vault_ledger (999_VAULT) as a SEAL verdict. The session token creates an immutable audit trail.
