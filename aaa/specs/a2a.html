A2A Protocol

arifOS MCP implements Google's Agent-to-Agent (A2A) protocol for inter-agent task delegation. Any A2A-compatible orchestrator can discover, authenticate, and delegate tasks to this runtime.

Agent Identity IDENTITY

The agent card is machine-readable JSON served at the well-known URI. A2A-compatible clients fetch this to discover capabilities before delegating tasks.

/.well-known/agent.json — Agent Identity
agent_id arifosmcp.arif-fazil.com
name arifOS MCP Runtime
version v2026.03.24
author Muhammad Arif bin Fazil
architecture Trinity ΔΩΨ
constitutional_floors 13
a2a_endpoint https://arifosmcp.arif-fazil.com/a2a
mcp_endpoint https://arifosmcp.arif-fazil.com/mcp

Capabilities CAPABILITIES

The arifOS runtime exposes the following capability categories to A2A orchestrators. Each capability maps to one or more of the 11 mega-tools internally.

reasoning
First-principles analysis, logical deduction, and cognitive synthesis via agi_mind (111/222).
red-teaming
Adversarial probing, counter-argument generation, and vulnerability surfacing via asi_heart (555_EMPATHY).
code-execution
Sandboxed Python execution for computations, data transforms, and proof-of-concept code via code_engine (333).
memory
Cross-session persistent memory storage and retrieval via engineering_memory (555_MEMORY).
reality-search
Qdrant vector search plus live web retrieval for grounded evidence via physics_reality (333).
governance
Constitutional scoring, floor traversal audit, and verdicts via apex_soul (777/888) and vault_ledger (999).
quantitative
Entropy budgets, Fermi estimation, and mathematical reasoning via math_estimator (222).
arbitration
Multi-stakeholder conflict resolution and constitutional scoring for high-stakes decisions.

Authentication AUTH

A2A authentication mirrors the MCP authentication model. The endpoint is open by default; key-gated when the server environment is configured.

MethodHeaderRequiredDescription
None — default Open access when COPILOT_API_KEY is not set on server.
API Key X-API-Key: <key> when configured Required when COPILOT_API_KEY env var is set on the runtime server.
Governance X-Governance-Secret: <secret> override only Required only for F13 Sovereign override operations.
Agent Authentication: When submitting tasks, include your agent's identity in the sender field of the task message. This is used for audit trails in vault_ledger but is not used for authorization.

Task Message Format SCHEMA

A2A tasks are submitted as JSON POST requests to the /a2a endpoint. The task message envelope wraps the actual task payload.

POST https://arifosmcp.arif-fazil.com/a2a
Content-Type: application/json

{
  "id":        "<task-uuid>",          // Unique task identifier
  "sender":    "<agent-id@domain>",   // Sending agent identity
  "recipient": "arifosmcp.arif-fazil.com",
  "timestamp": "2026-03-24T06:32:00Z",  // ISO 8601
  "task": {
    "type":    "<task-type>",           // See capabilities list
    "priority": "normal",               // low | normal | high | critical
    "input": {
      // Task-specific payload
    },
    "metadata": {
      "session_token":  "<tok from init_anchor>",  // Recommended
      "deadline_utc":   "2026-03-24T07:00:00Z",   // Optional
      "require_vault":  true                        // Log to vault_ledger
    }
  }
}
FieldTypeRequiredDescription
id string required UUID or unique task identifier. Echoed in all responses.
sender string required Sending agent's identity string (e.g. orchestrator@myagent.com).
recipient string required Must be arifosmcp.arif-fazil.com.
timestamp string required ISO 8601 UTC timestamp of task creation.
task.type string required One of: reasoning, red-teaming, code-execution, memory, reality-search, governance, quantitative, arbitration.
task.priority string optional Task urgency: low | normal | high | critical. Default: normal.
task.input object required Task-type–specific payload. See examples below.
task.metadata.session_token string recommended Session token from MCP init_anchor for audit continuity.
task.metadata.require_vault boolean optional If true, result is sealed in vault_ledger regardless of stakes level.

Task State Machine STATES

A2A tasks move through the following states. The current state is returned in every response envelope.

submitted → working → input_required → working → completed
StateMeaning
submittedTask received; queued for constitutional processing.
workingCurrently traversing constitutional floors (F01–F13).
input_required888_HOLD triggered; awaiting human confirmation before proceeding.
completedAll floors passed; verdict sealed (SEAL, SABAR, or VOID).
failedRuntime error or VOID verdict; task cannot be completed.

Response Format RESPONSE

All A2A responses share a common envelope. Synchronous tasks return the final result immediately; long-running tasks may be polled via the task status endpoint.

{
  "id":          "<task-uuid>",
  "status":      "completed",           // submitted | working | input_required | completed | failed
  "timestamp_utc": "2026-03-24T06:32:15Z",
  "result": {
    // Task-type specific output
  },
  "envelope": {
    "verdict":              "SEAL",
    "floor":                13,
    "constitutional_score": 0.94,
    "session_token":        "tok_xxx",
    "stage":                "999_VAULT",
    "flags":                [],
    "hold_reason":          null
  },
  "error": null                         // Populated on failure
}

Example — 888_HOLD Escalation F13 SOVEREIGN

When the pipeline determines that human confirmation is required before proceeding, the task enters input_required state and a 888_HOLD verdict is returned.

{
  "id":            "task-20260324-critical-007",
  "status":        "input_required",
  "timestamp_utc": "2026-03-24T06:33:05Z",
  "result":        null,
  "envelope": {
    "verdict":              "888_HOLD",
    "floor":                13,
    "constitutional_score": 0.61,
    "session_token":        "tok_2026_abc123xyz",
    "stage":                "888_JUDGE",
    "flags":                [
      "CRITICAL_STAKES",
      "IRREVERSIBLE_ACTION",
      "F13_SOVEREIGN_TRIGGERED"
    ],
    "hold_reason": "Action classified as irreversible with high societal \
impact. F13 Sovereign requires explicit human confirmation before \
vault_ledger sealing and execution authorisation."
  },
  "error": null,
  "hold_instructions": {
    "confirmation_endpoint": "https://arifosmcp.arif-fazil.com/a2a/confirm",
    "task_id":    "task-20260324-critical-007",
    "expires_utc": "2026-03-24T07:33:05Z",
    "instructions": "POST to /a2a/confirm with your human-authorised \
confirmation_token and task_id within 1 hour."
  }
}
888_HOLD behaviour: The task is frozen. No action is taken until the orchestrator receives explicit human authorisation and re-submits with a confirmation_token. HOLD states expire after 1 hour by default.
