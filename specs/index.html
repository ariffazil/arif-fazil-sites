arifOS — Protocol Specifications

These pages define how agents discover and communicate within the arifOS estate.

A2A — Agent-to-Agent Protocol

Standard: Agent2Agent by Google / Linux Foundation

Spec: https://a2a-protocol.org/latest/

Every agent in the arifOS Trinity publishes an Agent Card at /.well-known/agent.json.

arifOS Agent Cards

  • arif — THE SOUL (Ring 1, observer)
  • arifOS Kernel (Ring 2, advise)
  • AAA Runtime (Ring 3, execute)
  • Forge Portal (discovery)

MCP — Model Context Protocol

Standard: MCP by Anthropic / Model Context Protocol

Spec: https://modelcontextprotocol.io/specification

arifOS MCP Server: https://arifosmcp.arif-fazil.com/mcp

WebMCP — Browser-Native Tool Registry

Registered on pages with real interactive tools:

  • AAA Runtime — geox lookup, Archie saturation, porosity tools
  • GEOX Earth — 99-material atlas, well data explorer

How They Connect

A2A handles agent↔agent discovery. MCP handles agent↔tool calls. WebMCP lets browsers register tools directly.

All three are complementary. A2A agents discover each other. MCP tools execute. WebMCP exposes browser-side resources.
